# DR-020：机器人训练数据的模态与采集模式分析

## 动机

机器人训练数据集的多样性不只是"有哪些传感器"，还包括"怎么采集的"——遥操作、自主采集、仿真生成、人类演示视频等。不同采集模式产出的数据有不同的模态组合和分布特性。数据本身的结构可能直接影响训练出的表征质量。

## 任务

### 1. 主要机器人数据集的完整清单

对每个数据集列出：
- **传感模态**：RGB / depth / stereo / wrist camera / point cloud / proprio（关节角/速度/力矩）/ force-torque / tactile / language / eye-in-hand / extrinsic camera
- **采集模式**：遥操作（VR/键盘/spacemouse）/ 自主探索 / 仿真 / 人类演示视频（无本体）/ 脚本化 / 混合
- **有/无本体感觉**：纯视觉 vs 视觉+proprio vs 全模态
- **规模**（小时数/轨迹数/episode数）
- **机器人种类和数量**

包括但不限于：DROID、Open X-Embodiment、BridgeV2、ALOHA、RoboTurk、RoboNet、RH20T、CALVIN、RLBench、ManiSkill、MetaWorld、RoboMimic 等。

### 2. 采集模式对数据质量/分布的影响

- 遥操作 vs 自主 vs 仿真：数据分布有什么结构性差异？
- 无本体（如人类演示视频）的数据能提供什么信息？缺什么？
- 仿真数据和真实数据的模态差异——sim-to-real gap 在数据层面长什么样？
- 有没有人分析过不同采集模式的数据对训练出的表征的影响？

### 3. 模态组合对表征的影响

- 有没有模态 ablation 实验（去掉 depth / 去掉 proprio / 去掉 language 后性能/表征怎么变）？
- 加哪个模态对表征质量提升最大？有没有实证？
- 不同模态之间的信息冗余度——proprio 和 vision 编码了多少重复信息？

### 4. 数据本身的表征分析

- 有没有人对数据本身（不是模型的，而是原始数据）做过几何/统计分析？
- 不同模态之间的统计相关性？
- 不同采集模式产出的数据的流形结构有什么不同？

### 5. Cross-dataset 的模态不一致问题

不同数据集模态不统一（有的有 depth 有的没有、有的有 proprio 有的是纯视频）——混合训练时怎么处理的？有没有分析模态缺失对表征的影响？
